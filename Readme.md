

   ```bash
   npm run build
   ```

4. **Run linting**

## 🎨 Design System

### Component Library

Our application is built on [shadcn/ui](https://ui.shadcn.com/) components with custom theming and Amazon brand integration.

#### Available UI Components

| Component | Usage | Variants |
|-----------|-------|---------|
| **Button** | `<Button variant="default" size="lg">Click me</Button>` | `default`, `destructive`, `outline`, `secondary`, `ghost`, `link` |
| **Card** | `<Card><CardHeader><CardTitle>Title</CardTitle></CardHeader></Card>` | Includes `CardHeader`, `CardContent`, `CardFooter`, `CardTitle`, `CardDescription` |
| **Badge** | `<Badge variant="success">Status</Badge>` | `default`, `secondary`, `destructive`, `outline`, `success`, `warning`, `info` |
| **Input** | `<Input type="text" placeholder="Enter text" />` | Standard HTML input props |
| **Textarea** | `<Textarea placeholder="Enter message" />` | Standard textarea props |
| **Checkbox** | `<Checkbox checked={true} />` | Uses Radix UI primitive |
| **Progress** | `<Progress value={75} />` | Value from 0-100 |
| **Select** | `<Select><SelectTrigger>...</SelectTrigger></Select>` | Full Radix UI Select implementation |
| **Dialog** | `<Dialog><DialogTrigger>...</DialogTrigger></Dialog>` | Modal dialogs with backdrop |
| **Dropdown Menu** | `<DropdownMenu><DropdownMenuTrigger>...</DropdownMenuTrigger></DropdownMenu>` | Context menus and dropdowns |
| **Tabs** | `<Tabs defaultValue="tab1"><TabsList>...</TabsList></Tabs>` | Tabbed interface component |

### Component Patterns

#### 1. Variant-Based Design
Components use `class-variance-authority` for consistent variant management:

```tsx
const buttonVariants = cva(
  "base-classes",
  {
    variants: {
      variant: {
        default: "default-styles",
        outline: "outline-styles"
      },
      size: {
        default: "default-size",
        sm: "small-size"
      }
    }
  }
);
```

#### 2. Composition Pattern
Components are built for composition with consistent prop interfaces:

```tsx
interface ComponentProps
  extends React.HTMLAttributes<HTMLElement>,
  VariantProps<typeof componentVariants> {
  // Additional props
}
```

#### 3. forwardRef Pattern
All components use React.forwardRef for ref forwarding:

```tsx
const Component = forwardRef<HTMLElement, ComponentProps>((
  { className, variant, ...props },
  ref
) => {
  return <element ref={ref} className={cn(componentVariants({ variant }), className)} {...props} />;
});
```

## 🧪 Development Guidelines

### Creating New Components

1. **Use existing patterns**: Follow the variant-based design system
2. **Leverage Radix UI**: For complex interactive components
3. **Include TypeScript**: All components must be fully typed
4. **Test responsiveness**: Ensure mobile-first design
5. **Support theming**: Use CSS custom properties for colors

- **Formatting**: ESLint configuration enforced
- **Naming**: PascalCase for components, camelCase for props

- **Styling**: Tailwind classes with custom utilities when needed
